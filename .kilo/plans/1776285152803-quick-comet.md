# Plano de Correção: CRM WhatsApp - Recebimento e Processamento de Mensagens

## Diagnóstico

### Problemas Identificados

#### 1. **CRÍTICO: receive-message não encontra leads (404)**
**Causa Raiz**: A função `parseN8nPayload` extrai o telefone do lead de `body.chat.phone`, mas o payload UaZAPI coloca o telefone em `body.message.sender_pn`.

Payload UaZAPI real (de debug_payloads):
```json
{
  "chat": { "phone": "+55 21 97729-7413", ... },
  "message": {
    "sender_pn": "5521977297413@s.whatsapp.net",
    ...
  }
}
```

A função espera `body.chat.phone` → funciona quando presente, MAS:
- Para mensagens de **grupo** (`isGroup: true`), `body.chat.phone` está vazio
- O lead é encontrado pelo número do dono da conta (`owner`), não do remetente

**Evidência**: `debug_payloads` mostra que as mensagens do João Ferreira (21977297413) entram mas NÃO criam mensagens na tabela `mensagens` porque retornam 404 do `receive-message`.

#### 2. **ERRO "Cannot read image" no whatsapp-ai-agent**
**Causa Raiz**: Quando uma mensagem de **imagem** chega, ela é direcionada para o Grok-3 que não suporta input de imagem. O código JÁ TEM tratamento para isso (linha 259), mas há um bug no fluxo de error handling.

O problema específico:
- O loop de retry (linha ~247) só trata erros `503`
- Erros de imagem retornam como `400` ou `500`, não `503`
- O retry loop faz `break` prematuramente quando encontra erro não-503
- A mensagem de imagem com `[Imagem recebida]` é enviada ao Grok, que rejeita com "Cannot read image"
- A detecção do erro ocorre DEPOIS do loop, mas a condição `if (errorMsg.includes("image")...` pode não funcionar corretamente

#### 3. **Mensagens de grupo não são processadas**
**Causa**: `receive-message` não filtra mensagens de grupo (não deveria processar grupos), mas o webhook pode estar recebendo essas mensagens e retornando 200 sem processar.

## Soluções

### Correção 1: receive-message - parseN8nPayload (CRÍTICO)

**Arquivo**: `supabase/functions/receive-message/index.ts`

A função `parseN8nPayload` precisaextrair o telefone de `body.message.sender_pn` (ou `body.from`) quando `body.chat.phone` está vazio.

```typescript
function parseN8nPayload(payload: any) {
  const data = Array.isArray(payload) ? payload[0] : payload;
  const body = data?.body || data || {};
  
  // Tentar múltiplas fontes para o telefone
  const from = body.chat?.phone || body.message?.sender_pn?.replace('@s.whatsapp.net','') || body.from || null;
  if (!from) return null;
  
  // ... resto da função inalterado
}
```

### Correção 2: whatsapp-ai-agent - Tratamento de Imagem (CRÍTICO)

**Arquivo**: `supabase/functions/whatsapp-ai-agent/index.ts`

O erro "Cannot read image" acontece quando:
1. A mensagem do lead é uma imagem → texto vira `[Imagem recebida - a IA não suporta processamento de imagens]`
2. Esse texto é enviado ao Grok (mesmo sem imagem, o modelo Grok-3 pode rejeitar se detectar que o usuário enviou uma imagem em outra turn)
3. O erro "Cannot read image" é retornado

**Solução**: Modificar a lógica de retry para também capturar erros de imagem e tratá-los corretamente ANTES do loop de retry:

```typescript
// ANTES do loop de retry, verificar se é erro conhecido de imagem
const imageErrorPatterns = ["Cannot read image", "image", "vision", "visual"];
const isImageError = imageErrorPatterns.some(pattern => grokError?.message?.includes(pattern));

if (isImageError) {
  if (execLogId) await updateLog(execLogId, {
    status: "error",
    etapa: "erro_grok_imagem",
    erro_detalhe: `Grok não suporta imagens: ${grokError?.message}`,
    duracao_ms: Date.now() - globalStart,
  });
  return jsonResponse({
    ok: false,
    reason: "ia_nao_suporta_imagens",
    detalhe: "A IA configurada não suporta processamento de imagens."
  });
}
```

Ou seja, mover a detecção de erro de imagem PARA FORA do loop de retry, para que seja capturada imediatamente sem passar pelo retry.

### Correção 3: receive-message - Ignorar Mensagens de Grupo

**Arquivo**: `supabase/functions/receive-message/index.ts`

Adicionar verificação para ignorar mensagens de grupo (grupos não devem ser processados como leads):

```typescript
// Após parsear o payload, verificar se é mensagem de grupo
if (rawPayloadData?.isGroup || rawPayloadData?.chat?.wa_isGroup) {
  // Grupo: não processar como lead individual
  return new Response(JSON.stringify({ ok: true, skipped: 'group_message' }), { status: 200, headers: corsHeaders });
}
```
